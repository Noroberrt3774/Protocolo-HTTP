Protocolo HTTP

¿Qué significa HTTP?

HTTP son las siglas de HyperText Transfer Protocol, que en español quiere decir Protocolo de Transferencia de Hipertexto.

El HTTP es un conjunto de reglas que permite que los navegadores web (como Chrome, Firefox o Edge) y los servidores web se comuniquen entre sí. Es decir, cuando tú entras a una página web, tu navegador le pide información al servidor donde está guardada esa página, y el servidor se la envía usando HTTP.

Estructura y funcionamiento del protocolo HTTP

Funcionamiento básico

HTTP funciona bajo el modelo cliente-servidor:

  • Cliente: Generalmente un navegador web. Realiza una solicitud (request) al servidor.
  • Servidor: Recibe la solicitud, la procesa y devuelve una respuesta (response) al cliente.

Este proceso se hace generalmente sobre el protocolo TCP/IP, y con frecuencia se usa en conjunto con HTTPS, que es la versión segura de HTTP.

Estructura de una solicitud

Tiene tres partes principales

  • Línea de petición: Incluye el método como GET (para obtener datos) o POST (para enviar datos), la URL (La dirección a la que se está haciendo la solicitud) y la versión de HTTP.
  • Encabezados (headers): Información adicional como tipo de navegador, formatos aceptados, etc.
  • Cuerpo (body) (opcional): Este es el contenido que se envía al servidor, normalmente en peticiones POST. por ejemplo, datos de un formulario.

Estructura de una respuesta

Código de estado

Es un número de tres dígitos acompañado de una frase que indica si la solicitud fue exitosa o si hubo algún problema.

Se agrupan en cinco categorías:

  • 1XX: Informativos (por ejemplo, 100 Continuar)
  • 2XX: Éxito (ejemplo, 200 OK: la solicitud fue exitosa)
  • 3XX: Redirecciones
  • 4XX: Errores del cliente (ejemplo, 404 Not Found: recurso no encontrado)
  • 5XX: Errores del servidor (ejemplo, 500 Internal Server Error)

Encabezados

Contienen metadatos o información adicional sobre la respuesta.

Ejemplos de información en encabezados: Tipo de contenido que contiene la respuesta (como "text/html" o "application/json"). Longitud del contenido (número de bytes del cuerpo). Datos sobre el servidor que procesa la solicitud. Parámetros que controlan la caché o la conexión.

Cuerpo

  • Es la parte que contiene los datos solicitados.
  • Puede incluir páginas web HTML, datos JSON, imágenes, videos, u otros archivos.
  • La ausencia de cuerpo es posible en ciertas respuestas, como cuando el código de estado es 204 (sin contenido).
